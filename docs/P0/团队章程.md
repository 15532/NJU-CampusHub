# CampusHub 团队章程与工具选型

## 1. 角色分工
* **架构/开发负责人 ([你的名字]):** 负责主导前后端分离技术栈选型与系统整体架构设计。制定接口规范，集成 AI 辅助编码工具并统筹核心模块开发与代码审查。
* **需求负责人 ([成员B名字]):** 负责需求获取、文档撰写、用例建模，把控平台核心业务流转逻辑。
* **测试负责人 ([成员C名字]):** 负责测试策略、测试用例设计、质量保障，拦截潜在系统 Bug。
* **[其他角色] ([成员D名字]):** [填写其职责]

## 2. AI 工具链选型
* **AI 编程助手:** GitHub Copilot (与主流 IDE 无缝集成，上下文补全能力强)
* **AI 对话工具:** Gemini / Claude 3.5 (擅长处理长文档分析、系统架构设计推演)
* **UI 协同生成:** Google Stitch (快速生成高保真 React 规范前端代码，契合 Taro 架构)
* **代码审查工具:** CodeRabbit (深度集成 GitHub PR，自动进行逐行 Review 和安全扫描)
* **AI 测试工具:** CodiumAI (自动生成 JUnit 边界测试用例，节省手写时间)
