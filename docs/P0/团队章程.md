## 2.CampusHub 团队 AI 工具链选型分析报告

### 1. AI 编程助手 (AI Coding Assistants)

日常编码、代码补全、逻辑解释，需深度集成至现有开发环境。

#### 候选 1：GitHub Copilot (底层基于 Codex) —— 【选择】
* **功能：** 行业标杆级别的上下文理解与代码生成能力，完美支持 Java (Spring Boot) 等主流技术栈。
* **易用性：** 与 IntelliJ IDEA、WSL2 环境无缝集成，沉浸式开发体验极佳。
* **成本：** 团队中已有一位成员拥有 Codex/Copilot 权限，该成员可将其作为主力工具负责核心架构开发，最大化利用现有沉没成本。
* **团队熟悉度：** 高。开发者对插件形态的辅助工具接受度极高。

#### 候选 2：Cursor (AI 原生 IDE) —— 【不选择】
* **功能：** 多文件协同编辑和代码重构能力极强。
* **易用性：** 作为独立 IDE，操作逻辑基于 VS Code。
* **成本：** 免费额度有限，高频使用需额外订阅。
* **团队熟悉度及不选择理由：** 团队已经建立了成熟的 IntelliJ IDEA 等环境配置和使用习惯，整体迁移至 Cursor 需要一定的适应期。为了保持开发环境的一致性和稳定性，暂不引入新的独立 IDE。

---

### 2. AI 对话工具 (AI Chat Tools)

用于长文档阅读（如第三方接口文档）、需求分析、系统架构设计推演及撰写开发文档。

#### 候选 1：Gemini Pro / Claude 3.5 —— 【选择】
* **功能：** 具备超大上下文窗口（Gemini）和顶尖的系统架构推演能力（Claude 3.5），特别擅长处理复杂的长文档分析和多模块逻辑 Debug。
* **易用性：** Web/App 端随时可用，交互自然。
* **成本：** 团队**全员已拥有 Gemini Pro**。
* **团队熟悉度：** 极高。全员已有使用习惯，能迅速投入到需求拆解和 LaTeX/Markdown 文档生成中。

#### 候选 2：ChatGPT Plus (GPT-4o) —— 【不选择】
* **功能：** 综合能力处于第一梯队，插件生态丰富。
* **易用性：** Web 端体验良好。
* **成本：** 需要每人每月 20 美元的订阅费。
* **不选择理由：** 团队现有的 Gemini Pro 在长文本处理和架构推演上已完全能满足项目需求，引入 ChatGPT Plus 会导致功能重叠并大幅增加团队财务负担。

---

### 3. AI 代码审查工具 (AI Code Review Tools)

集成在 Git 工作流中，规范 4 人协作的代码风格，拦截潜在错误。

#### 候选 1：CodeRabbit —— 【选择】
* **功能：** 自动逐行进行 Code Review，提供上下文感知的建议，并自带安全扫描功能。
* **易用性：** 深度集成于 GitHub PR 流程中，配置简单，完全自动化触发。
* **成本：** 若项目代码托管在公共仓库，可免费使用，性价比极高。
* **团队熟悉度及选择理由：** 虽然团队初次接触自动化 Review 工具，但其开箱即用的特性几乎不需要学习成本，能极大提升团队代码合并时的质量保障。

#### 候选 2：自建 CI/CD 脚本调用大模型 API —— 【不选择】
* **功能：** 通过编写脚本，在代码提交时抓取 diff 并调用 AI 接口进行评审，规则高度可定制。
* **易用性：** 需要前期投入时间编写 Webhook 和处理逻辑，维护成本较高。
* **成本：** API 调用成本低（可复用现有的 Gemini Pro 额度）。
* **不选择理由：** 对于时间紧凑的校园互助平台项目，敏捷开发是第一要务。造轮子虽然能锻炼能力，但不如直接使用成熟商业工具 CodeRabbit 来得高效。

---

### 4. AI 测试工具 (AI Testing Tools)

自动生成测试用例，保障核心业务（如互助订单匹配流转）的稳定性。

#### 候选 1：CodiumAI (Qodo Gen) —— 【选择】
* **功能：** 能够分析代码行为，自动生成覆盖正常路径和边缘边界条件的 JUnit 测试用例。
* **易用性：** 以插件形式无缝接入主流 IDE，可在编写完业务逻辑后一键生成测试脚手架。
* **成本：** 基础版本免费，满足绝大多数单测生成需求。
* **团队熟悉度及选择理由：** 能显著减少手写繁琐测试断言的时间，让测试负责人的精力集中在整体测试策略和集成测试上，对提升工程规范极有帮助。

#### 候选 2：Diffblue Cover —— 【不选择】
* **功能：** 针对 Java 生态的重型自动化测试全覆盖工具。
* **易用性：** 配置相对繁琐，需要较深的测试体系集成。
* **成本：** 偏向企业级付费工具。
* **不选择理由：** 工具过于笨重，配置门槛高，不符合 4 人学生团队追求敏捷迭代和轻量级开发的节奏。
