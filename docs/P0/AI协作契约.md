# 《AI 协作契约》

团队名称：CampusHub 开发组
签署日期：2026年4月16日

**第一条：AI 使用原则**
- AI 是工具，工程师是责任主体。
- 所有 AI 输出必须经过人工审查后才能合入项目。

**第二条：AI 使用范围**
- **允许使用：** 样板代码 (CRUD) 生成、正则/SQL 语句编写、基础测试用例起草、纯前端布局代码 (HTML/JSX + Tailwind) 生成。
- **禁止直接使用 (需额外审查)：** 核心状态机逻辑（如订单接单到完成的流转）、前端状态管理 (Hooks)、涉及密码加密的鉴权模块。
- **完全禁止：** 架构决策的盲目听从、未经验证的第三方依赖库引入。

**第三条：过程记录要求**
- 所有与 AI 的交互 Prompt 必须保留日志。
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]。
- 每阶段填写《AI 协作反思日志》。

**第四条：代码合入规则**
- AI 生成的代码必须通过至少一名团队成员的 Code Review。
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数。

**第五条：违约处理**
- 未经 Review 直接合入 AI 生成代码导致主分支崩溃者，负责当周组会请客喝奶茶并修复 Bug。
